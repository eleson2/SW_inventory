## Documentation Standards

### Writing Guidelines

1. **Be Clear**: Use simple, direct language
2. **Be Complete**: Cover all aspects of the topic
3. **Be Current**: Update docs when code changes
4. **Be Practical**: Include real examples
5. **Be Helpful**: Anticipate user questions

### Documentation Format

- **Markdown**: All documentation uses GitHub-flavored Markdown
- **Code Blocks**: Include language identifiers for syntax highlighting
- **Links**: Use relative links between documentation files
- **Images**: Store in `docs/images/` (when added)

### Maintaining Documentation

When making changes to the codebase:

1. **Update Relevant Docs**: If you change functionality, update the docs
2. **Add Examples**: Include code examples for new features
3. **Update Screenshots**: If UI changes, update screenshots (when added)
4. **Version Notes**: Document breaking changes
