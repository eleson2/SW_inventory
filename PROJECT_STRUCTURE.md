patterns:\n- **Button**: Multiple variants (default, secondary, destructive, outline, ghost, link) and sizes\n- **Card**: Container component for content grouping\n- **Input**: Form input with error state support\n- **Label**: Form label with required indicator\n- **Badge**: Tag/label component for statuses and versions\n- **Table**: Basic table wrapper with overflow handling\n\n### Common Components (`$components/common/`)\nApplication-specific reusable components:\n- **DataTable**: Full-featured table with sorting, pagination, and row click handling\n- **FormField**: Complete form field combining Label + Input + error display\n- **Pagination**: Page navigation with smart ellipsis handling\n- **StatusBadge**: Standardized active/inactive status display\n\n### Domain Components (`$components/domain/`)\nBusiness domain-specific components:\n- **VersionDisplay**: Formats and displays software versions with PTF levels\n\n## Key Features & Utilities\n\n### Version Parsing (`$utils/version-parser.ts`)\n- Parse vendor designations into structured version objects\n- Support for multiple version formats (V2R4M0-PTF12345, 2.4.0 (PTF 12345), etc.)\n- Version comparison logic for versions and PTF levels\n- Compatibility checking for package requirements\n\n### Package Service (`$lib/services/package-service.ts`)\n- Calculate customer-specific package subsets\n- Validate LPAR package compliance\n- Generate deployment plans for package upgrades\n- Handle software rollback logic\n- Calculate compatibility scores\n\n### Validation Schemas (`$schemas/`)\nComprehensive Zod schemas for all entities:\n- Customer, Vendor, Software, Package, LPAR schemas\n- Create and update variants\n- Pagination, sorting, and filtering schemas\n- Form validation with detailed error messages\n\n## Design Patterns\n\n### Component Design\n- Svelte 5 runes syntax (`$props`, `$state`, `$derived`)\n- TypeScript for type safety\n- Consistent prop naming conventions\n- Reusable render props pattern for flexible rendering\n\n### Form Handling\n- SvelteKit form actions for server-side processing\n- Zod validation schemas\n- Comprehensive error handling and display\n- Consistent form field structure\n\n### Data Loading\n- SvelteKit `load` functions in `+page.server.ts`\n- Type-safe PageData with `$types`\n- Pagination and sorting support\n- Mock data structure for easy database integration\n\n### Styling\n- Tailwind CSS for utility-first styling\n- CSS custom properties for theming\n- Consistent color palette (primary, secondary, destructive, etc.)\n- Responsive design utilities\n- Dark mode support (theme variables included)\n\n## Next Steps for Implementation\n\n1. **Database Integration**\n   - Add Prisma/Drizzle ORM\n   - Replace mock data in `+page.server.ts`
